Go日志库zap使用

2019-12-16 6 分钟阅读 go三方库使用和源码分析go

zap 是一个高性能的第三方日志类库，本篇主要说明下如何使用 zap

版本

本篇基于的 zap 版本是 v1.10.0

使用

基本概念

在使用 zap 之前，我们还是先了解下 zap 里面的一些基本概念，方便在我们阅读 demo 或者写代码的时候能大体的感知到我们在用什么，为什么这样用。为了不产生翻译上的歧义，书写过程中涉及到的概念会用 zap 源码中的命名

Logger

这是 zap 暴露给我们类型，正常情况下，我们也是通过 Logger 这个结构体的方法来记录日志的

Config

这个是 zap 提供给我们的 Logger 的配置，我们可以按需配置，然后通过 Build 方法来构造出我们想要的 Logger，是典型的的构造者模式

Field

除了日志级别、日志时间、日志信息等这些基本的通用字段之外，我们在记录日志的时候通常会想记录一些额外的字段（用来记录打日志时的上下文），这些字段在 zap 中需要封装成 Field 类型

Core

Logger 是一个结构体，为我们暴露了使用日志的方法，而 Core 是一个接口，它被包含在 Logger 中，用来真正提供日志格式化和输出功能，这样的实现使日志记录器的使用和输出实现了高度解耦。我们可以实现自己的 Core 从而达到不同的输出的目的

Encoder

Encoder 同样是一个接口，刚才说到 Core 是实现具体输出行为的核心，而 Encoder 是 Core 中真正进行日志信息和字段格式化的编码器，它把我们输入的各种日志信息经过处理得到字节切片。从而可以记录在各种存储载体中。zap 对于 Encoder 自带了两个实现，jsonEncoder 和 consoleEncoder

EncoderConfig

这个就很好理解了，就是我们 Encoder 的配置，由于不同的 Encoder 实现不一样，所以如果是我们自己实现的 Encoder，完全可以实现自己的 EncoderConfig，zap 中的 EncoderConfig 自然也是为了 jsonEncoder 和 consoleEncoder 服务的。

Logger 快速入门

了解了 zap 中的一些基础概念，让我们先来搞一个小 demo 感受一下

代码：

 1func main() {
 2	// 实例化一个 EncoderConfig
 3	encoderCfg := zapcore.EncoderConfig{
 4		MessageKey:     "msg", // 序列化成json时在输出中的key
 5		LevelKey:       "level",
 6		NameKey:        "logger",
 7		EncodeLevel:    zapcore.LowercaseLevelEncoder, // 日志级别字段的值序列化方式，这里使用了zap自带的小写格式编码
 8		EncodeTime:     zapcore.ISO8601TimeEncoder,    // 时间字段的值序列化方式，这里使用了自带的ISO8601格式编码
 9		EncodeDuration: zapcore.StringDurationEncoder, // 时间段类型的值序列化方式，这里使用自带的字符串格式
10	}
11
12	// 用刚才 EncoderConfig 实例化一个 jsonEncoder，也就是输出成json格式
13	jsonEncoder := zapcore.NewJSONEncoder(encoderCfg)
14
15	// 实例化一个 Core，使用 jsonEncoder，将 InfoLevel 级别以上（包含 InfoLevel）的日志输出到标准输出中
16	core := zapcore.NewCore(jsonEncoder, os.Stdout, zapcore.InfoLevel)
17
18	// 用 Core 实例化一个 Logger
19	logger := zap.New(core)
20
21	logger.Debug("debug level log", zap.String("k", "v"))
22	logger.Info("info level log", zap.Bool("k", false))
23	logger.Warn("warn level log", zap.Time("t", time.Now()))
24	logger.Error("error level log", zap.Int("i", 9))
25}

输出：

1{"level":"info","msg":"info level log","k":false}
2{"level":"warn","msg":"warn level log","t":"2019-12-16T23:58:53.324+0800"}
3{"level":"error","msg":"error level log","i":9}

之所以少一条，是应为我们设置了 info 级别以上才输出。

使用构造方法获取预置 Logger

可以看到上面的整个使用过程，基本上都是围绕着我们最开始提到的基本概念展开的，同时也能明显的感受到，构造一个 Logger 的过程还是复杂的，所以同很多包一样，zap 也为我们提供了几个方便的构造函数，能让我们快速的获取 Logger，基本也能覆盖我们大多数简单的使用场景，分别是zap.NewExample，zap.NewDevelopment和zap.NewProduction，除了这3个，还有一个zap.NewNop，这个相当于一个空 Logger，不做任何操作，所以也不用着重介绍了。下面让我们依次看下前三个。

zap.NewExample

1func main() {
2	logger := zap.NewExample()
3	logger.Debug("我是debug信息", zap.String("k", "v"))
4	logger.Info("我是info信息", zap.Bool("b", true))
5}
6
7// {"level":"debug","msg":"我是debug信息","k":"v"}
8// {"level":"info","msg":"我是info信息","b":true}

可以看到这样用起来就非常方便了，如果我们取看他的源码，会发现其实就是封装了我们上面创建一个 Logger 的过程

zap.NewDevelopment

1func main() {
2	logger, _ := zap.NewDevelopment()
3	logger.Debug("我是debug信息", zap.String("k", "v"))
4	logger.Info("我是info信息", zap.Bool("b", true))
5}
6
7// 2019-12-17T11:40:33.606+0800    DEBUG   zap/main.go:9   我是debug信息   {"k": "v"}
8// 2019-12-17T11:40:33.606+0800    INFO    zap/main.go:10  我是info信息    {"b": true}

zap.NewProduction

1func main() {
2	logger, _ := zap.NewProduction()
3	logger.Debug("我是debug信息", zap.String("k", "v"))
4	logger.Info("我是info信息", zap.Bool("b", true))
5}
6
7// {"level":"info","ts":1576554152.8352141,"caller":"zap/main.go:10","msg":"我是info信息","b":true}

通过配置构造 Logger

上面的几种用法都是我们手动的创建 Logger，我们还可以通过构造 Logger 的 Config，从而来构建 Logger，可以通过 zap.NewDevelopment 的源码学习

这种方法也能给我们带来比较大的灵活性，能在 Logger 的配置上有更大的自由度

如果想更加精细的配置 Logger，比如同时使用多个 Core 输出，可以参考本文的 zap 高级配置小节

使用 SugaredLogger 进行日志记录

即便是 zap 为我们提供了几个预置的 Logger，但是当我们想添加自己的附加字段的时候，需要对每个字段调用对应的封装函数将其包装为 zap.Field 类型，这样做的好处是效率非常高，但是当我们不在乎这一点性能的提升，愿意用一点点性能的损耗来换取编程的方便的时候，可以使用 zap 为我们提供的 SugaredLogger 这个类，名字也非常形象，语法糖，方便我们的编程过程

SugaredLogger 是对 Logger 的封装，提供了更加方便的api供我们使用，其实就是不必去调用 zap 的一系列函数，将我们的额外字段封装成 Field 类型，在 SugaredLogger 底层会自动帮我们处理

Logger 和 SugaredLogger 之间的转换也非常简单，我们完全可以在一个项目中根据需求混合使用两种日志记录方式

 1func main() {
 2	logger, _ := zap.NewDevelopment()
 3
 4	// 通过 Logger 转换成 SugaredLogger
 5	sugar := logger.Sugar()
 6	sugar.Info("sugar info message")
 7	sugar.Infof("sugar info formatted message %s", "hehe")
 8	sugar.Infow("sugar info message with extra context", "key1", "val1", "ket2", true)
 9
10	// 通过 SugaredLogger 转换成 Logger
11	lg := sugar.Desugar()
12	lg.Debug("我是debug信息", zap.String("k", "v"))
13	lg.Info("我是info信息", zap.Bool("b", true))
14}
15
16//2019-12-17T12:08:14.535+0800    INFO    zap/main.go:12  sugar info message
17//2019-12-17T12:08:14.535+0800    INFO    zap/main.go:13  sugar info formatted message hehe
18//2019-12-17T12:08:14.535+0800    INFO    zap/main.go:14  sugar info message with extra context   {"key1": "val1", "ket2": true}
19//2019-12-17T12:08:14.535+0800    DEBUG   zap/main.go:18  我是debug信息   {"k": "v"}
20//2019-12-17T12:08:14.535+0800    INFO    zap/main.go:19  我是info信息    {"b": true}
