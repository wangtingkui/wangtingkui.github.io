php 中的匿名函数和 Closure

  2020-06-24   php基础   php Read Count: 2minutes

php 中匿名函数也叫做闭包函数，允许我们创建一个没有名称的函数，经常用作回调函数

php 中的匿名函数是基于 Closure 这个类来实现的

基本使用

通常会将匿名函数用作回调函数

array_filter([1,2,3], function ($item) {
    return $item % 2 == 0;
});

// 也可以赋值给变量，供后续使用
$func = function () {
    echo "called" . PHP_EOL;
}

$func();

使用父作用域变量

接触过其他有闭包特性语言的同学，一定会知道闭包的一大特点是可以使用父作用域的变量，php中的必报同样可以，但是使用上稍微有些区别

php 中使用父作用域的变量，需要使用 use 关键字传递

$a = 1;
$func = function () use ($a) {
    echo $a;
};
$func(); // 1
$a = 2;
$func(); // 1

可能有的同学会感到奇怪，为什么上面两次调用输出的都是1，这是因为在 php 中，传值方式默认使用的都是值传递，如果需要在匿名函数中保有对父作用域变量的引用，需要使用引用传值的方式

$a = 1;
$func = function () use (&$a) {
    echo $a;
};
$func(); // 1
$a = 2;
$func(); // 2

特别需要注意的是

  • 闭包的父作用域指的是定义闭包时的作用域，而不是调用时的父作用域
  • use 中传递的参数，不能是 $this 和 超全局变量

$this 的自动绑定

当闭包定义在类中的时候，可以在闭包函数内使用 $this 代表类的对象

<?php

class A
{
    protected $a = 1;

    public function test()
    {
        $func = function () {
            echo $this->a;
        };
        $func();
    }
}

(new A())->test(); // 1

如果不想自动绑定 $this，可以将闭包声明为静态的

<?php

class A
{
    protected $a = 1;

    public function test()
    {
        $func = static function () {
            var_dump($this); // null
        };
        $func();
    }
}

(new A())->test(); // 1

这样就获取不到 $this 了
